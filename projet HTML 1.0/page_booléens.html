Les booléens

Voici les exercices du chapitre fonctions et leur correction :

Si quelqu'un vous dit que "4 est égal à 5", vous lui répondez que "c'est faux".
Si maintenant la même personne vous dit que "7 est égal à 7", vous lui répondrez que "c'est vrai".
En Python, ces deux «affirmations» ("4 est égal à 5" et "7 est égal à 7") s'écriront "4 == 5" et "7 == 7" (notez bien le double signe égal).

Exercice 20 :

Affichez la valeur de l’expression :

4 == 5

En Python, "4 == 5" est appelé une expression.

Correction:
Le résultat renvoyé est False.

Pour l'instant nous avons vu deux grands types de données : les nombres (entier ou flottant) et les chaînes de caractères, il existe un troisième type tout aussi important que les deux premiers : les booléens.
Un booléen est un type de données (bool en Python) qui ne peut prendre que deux valeurs : vrai ("True") ou faux ("False"). Une expression (comme par exemple "4 == 5") est soit True, soit False.

Attention : notez le double égal "==" qui permet de distinguer une expression et une affectation (référencement d'une valeur par une variable). Le fait de confondre le "simple égal" et le "double égal" est une erreur classique qu'il faut éviter.

Exercice 21 :

Soit le programme suivant :

a = 4
b = 7

Quel est le résultat attendu après l'exécution de ce programme si vous affichez la valeur de "a==b" ?

Correction:
Le résultat renvoyé est False.

Il est possible d'utiliser aussi l'opérateur "différent de" : " != "

Exercice 22 :

Soit le programme suivant :

a = 4
b = 7

Quel est le résultat attendu après l'exécution de ce programme si vous affichez la valeur de "a!=b" ?

Correction:
Le résultat renvoyé est True.

Notez aussi l'existence des opérateurs :

  • "strictement inférieur à : " < "
  • "strictement supérieur à : " > "
  • "inférieur ou égal à : " <= "
  • "supérieur ou égal à : " >= "
À chaque fois les valeurs des expressions utilisant ces opérateurs sont True (vrai) ou False (faux).

Exercice 23 :

Soit le programme suivant :

a = 4
b = 7

Quel est le résultat attendu après l'exécution de ce programme si vous affichez la valeur de "a < b" ?

Correction:
Le résultat renvoyé est True.
